A monotonicity analysis of organizational tasks

01 — The idea

Meetings, approvals, status updates, reviews. Organizations coordinate by default.

But there's no formal rule for when coordination is actually required.

Distributed systems theory has a rule, and it maps straight onto organizations. The same test that says when concurrent processes need to coordinate says when people and teams do.

A task is monotonic when every step only adds to the result and never takes anything back. A monotonic task runs correctly with no coordination at all. It turns non-monotonic the moment new information can invalidate a conclusion that was already valid.

The key question is: Can new work validate old conclusions? If no, coordination is optional. If yes, it is required.
